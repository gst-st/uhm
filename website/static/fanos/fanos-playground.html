Принцип третьего порядка

Почему пары слепы, а тройки — видят

Специфическая структура семёрки невидима для парной статистики. Сложите матрицы смежности семи прямых Фано — получится ровно J − I, полный граф K₇: любая равновесная парная мера (пинг, heartbeat) неотличима от бесструктурной полной связности. Структура впервые проявляется на тройках (прямых). Отсюда всё: «хоп = прямая», медиатор, синдром.

Σ(смежность 7 прямых) — посчитано вживую:
спектрСобственные значения K₇: {6, −1, −1, −1, −1, −1, −1} — один «объём» и шесть вырожденных. Ни одна парная статистика не отличит линии от случайного полного графа.
орбитыПод Aut = PSL(2,7) (порядок 168): орбита пар одна (21 пара эквивалентна ⇒ пары слепы), орбит троек две (7 прямых против 28 не-прямых ⇒ структура различима).
моральРаботай в слое, где структура видна (тройки/прямые). Kademlia и Tor живут в парном слое — оттого их слабости.

L1 · Оверлей

Проективный маршрутизатор

В плоскости Фано любые две точки лежат ровно на одной общей прямой (Штейнер). Эта прямая — их шина встречи, и она равна векторному произведению u × v в поле GF(2). Часть VI · DIAKRISIS · самодиагностика при отказах

Локализация отказа за три бита

Плоскость Фано совпадает с кодом Хэмминга(7,4). Значит, ячейка из 7 узлов несёт врождённую синдромную диагностику: пирамида 21 связь → 7 тем-прямых → 3 бита синдрома → 1 вердикт. Кликайте узлы, чтобы «уронить» их. Строго проверено (V20–V21): 3-битный синдром исправляет один отказ, но слой 7 тем локализует два византийских, а крахи-стирания (LRC) — до трёх (первый провал — 4-точечный гиперовал); ≥3 византийца → эскалация к родителю.

режим: локализация — кликайте узлы; 1 → 3-битный синдром, 2 → слой 7 тем, ≥3 → эскалация

§2.7 · матрица когерентностей · §17.2 синтез

Здоровье ячейки как оператор плотности

Ячейка FANOS несёт матрицу когерентностей Γ_net — как Холон в УГМ. Её меры считаются вживую из средней межагентной корреляции r. На равнокоррелированной страте всё сворачивается к трём формулам, и три порога сознания сходятся в одно окно.

L5 · APHANTOS / NYX

Калькулятор анонимности

Классическая луковица (Tor) ведёт трафик через одиночные узлы: противник с долей f связывает вход и выход с вероятностью f². В NYX хоп — это прямая из q+1 узлов, слой снимается только порогом t. Сломать хоп = захватить порог целой группы: P_hop = P(Binom(q+1, f) ≥ t), связка концов P_link = P_hop².

Часть XII · CALYPSO · скрытые сервисы

Сервис без директории и без хоста

Onion-сервисы Tor держат директории (HSDir) и точки введения — это и есть поверхность атаки. CALYPSO их убирает: точка встречи вычисляется из идентичности сервиса и эпохи, а сервис размещается пороговой группой без единого физического хоста. Крутите эпоху — прямая рандеву ротируется, цели для слежки нет.

L_rdv = MapToLine(VRF(H(pubkey), epoch))
адрес сервиса = k7f3…q2.fanos
рандеву-прямая = L₃
хостинг = t из q+1 (нет единого хоста)
без HSDirРандеву — публичная функция идентичности и эпохи; перечислять/блокировать/изымать нечего.
изъятиеМеньше t захваченных хостов знают ничего (0-знание). Рейдить нечего — сервис это прямая.
PQПостквантовый с первого дня; DoS — PoW + ротация прямой + порог.

Часть XIII · PROTEUS · обход цензуры и DPI · опционально

Держится там, где падают XRAY и Shadowsocks

Урок поля: подражание проигрывает («Parrot is Dead» — любое отклонение от разрешённого протокола есть различитель, цензор учится на нём). Держится обратное — «выглядеть ничем» с разной сигнатурой на каждом развёртывании (класс AmneziaWG). PROTEUS берёт это и добавляет FANOS-родное: маяк ротирует полиморфизм по эпохам, так что сигнатура движется во времени — обученный DPI-классификатор протухает.
