# 02 · Спецификация ядра

> *Одна библиотека, переиспользуемая для человека, команды и агента. Она берёт оценённое состояние и его неопределённость и возвращает отсчёт с пробросанной неопределённостью. Всё, что нужно доменному модулю, он получает вызовом ядра; ядро не вызывает ничего, кроме линейной алгебры.*

## §1. Представление Γ {#представление}

Единственный входной тип ядра — **объект-оценка** `GammaEstimate`:

- `gamma`: комплексная эрмитова матрица $7\times7$, PSD, единичный след — порядок осей фиксирован как $[A,S,D,L,E,O,U]$.
- `cov`: ковариация над 48 вещественными степенями свободы (7 населённостей − 1 след + 21 величина + 21 фаза; ограничение следа убирает одну степень свободы населённости), используется для проброса доверия. Может быть `null` для якоря III.
- `anchor`: одно из `MEASURED | AUTOEPHEMERIS | ORACLE` (тег честности).
- `t`: временная метка, для сборки траектории.

**Валидация и проекция.** Всякая входящая `gamma` проецируется на ближайшее валидное состояние перед использованием: эрмитизация ($\Gamma \leftarrow \tfrac12(\Gamma+\Gamma^\dagger)$), обнуление отрицательных собственных значений, ренормировка следа. Расстояние проекции записывается как флаг качества входа — большая проекция означает, что оценщик произвёл нечто далёкое от физического состояния, что слой честности выносит наружу.

## §2. Одиннадцать инвариантов — точные процедуры {#инварианты}

Всякая процедура — чистая функция `gamma` (плюс, где отмечено, второе состояние для инкрементов). Все $O(1)$ по размеру задачи (фиксированные $7\times7$); единственная нетривиальная численность — эрмитово собственное разложение и один малый гессиан.

**Чистота $P$.** `P = trace(gamma @ gamma).real`. Диапазон $[1/7, 1]$; $1/7$ — максимально смешанное состояние «тепловой смерти» $I/7$, $1$ — чистое. Порог жизнеспособности $P > 2/7$ ([Т](/docs/core/dynamics/viability)).

**Рефлексия $R$.** `R = 1.0 / (7*P)`. Диапазон $[1/7, 1]$; нормированное расстояние от $I/7$. Полоса сознания $R \geq 1/3$ ([Т](/docs/consciousness/foundations/self-observation#формы-r)). Это *каноническая* R; не путать с $R_\varphi$ (ниже) — это различные рабочие формы, и ядро экспонирует обе под своими именами.

**Качество самомодели $R_\varphi$.** Требует карту самомодели $\varphi$. Для канонической (диссипативной) семьи `phi(gamma) = (1-k)*gamma + k*eye(7)/7` с `k = 1 - 1/(7*P)`; тогда `R_phi = 1 - fro(gamma - phi(gamma))**2 / fro(gamma)**2`. Диапазон $[0,1]$. Где доступна обученная цель $\rho_\theta$ ([двухмасштабная модель](/docs/proofs/categorical/formalization-phi#механизмы-rφ)), вместо неё используется `phi_theta(gamma) = (1-k)*gamma + k*rho_theta` — это позволяет практике поднимать базовую линию.

**Интеграция $\Phi$.** Вычисляется из HS-проекционного функционала [dimension-U](/docs/core/structure/dimension-u#мера-интеграции-φ): утрата информации при наилучшем факторизованном приближении Γ по минимально-информационному разбиению. Алгоритм: для всякого разбиения семи осей формируется приближение произведением маргиналов, берётся HS-расстояние до Γ; $\Phi$ — минимум по разбиениям («слабейшее звено»), нормированный так, что порог сознания стоит на $\Phi = 1$. Шестьдесят три разбиения — тривиально перечисляемы.

**Дифференциация $D_{\mathrm{diff}}$.** `evals = eigvalsh(gamma); S = -sum(evals * log(evals + eps)); D_diff = exp(S)`. Диапазон $[1, 7]$; эффективное число занятых измерений. Порог $D_{\mathrm{diff}} \geq 2$ ([Т](/docs/proofs/consciousness/operationalization)).

**Карта Gap.** Для всякой пары $(i,j)$: `Gap[i,j] = abs(sin(angle(gamma[i,j])))`, с [конвенцией исчезающей когерентности](/docs/core/dynamics/gap-operator#конвенция-нулевой-когерентности): если `abs(gamma[i,j]) < eps_min`, положить `Gap[i,j] = 1` (канал без когерентности не «прозрачен», он отсутствует — фаза не определена). Возвращает 21 значение верхнего треугольника как карту прозрачности.

**Мандалаграмма.** Чистое переформатирование: 7 населённостей `diag(gamma).real` и 21 когерентность `(величина, фаза)` на карте 7-узлов / 21-ребро, с [каноническими именами ячеек](/docs/applied/research/gamma-canon#слой-1). Никаких вычислений, только раскладка — это render-модель, которую потребляет UI.

**Архетип.** Пороговать диагональ на $\tau$ (по умолчанию средняя населённость) до 7-битного паттерна; декодировать в одну из [шестнадцати сигнатур](/docs/applied/research/gamma-canon#слой-4) фиксированным поиском (16 сигнатур — $G_2$-орбитные классы пороговых диагоналей). Возвращает id сигнатуры плюс отступ до ближайшей соседней сигнатуры (насколько робастна классификация).

**Мода.** Требует два состояния $\Gamma(t)$, $\Gamma(t')$. Формируется инкремент $\dot\Gamma \approx (\Gamma(t')-\Gamma(t))/(t'-t)$ и раскладывается на [триадные компоненты](/docs/core/operators/lindblad-operators#триадная-декомпозиция) — гамильтонову (сохраняющую), диссипативную (растворяющую к $I/7$), регенеративную (восстанавливающую к $\varphi(\Gamma)$). Возвращает три доли, суммирующиеся в единицу; «мода» — доминирующая доля.

**Смысл.** `Meaning = P * D_diff * Phi * R_phi` — [произведённая форма](/docs/consciousness/ethics-meaning/meaning). Возвращается и как скаляр, и как четыре его фактора, чтобы UI мог показать, *какой* фактор ограничивает смысл (наименьший фактор — узкое место).

**Свобода.** Построить гессиан свободной энергии $\mathcal H_\Gamma$ (вторая производная функционала свободной энергии в Γ по 48 вещественным степеням свободы), взять его собственные значения, сосчитать те, что ниже допуска плоскостности: `Freedom = count(abs(hessian_evals) < tol) + 1` ([Т](/docs/consciousness/ethics-meaning/freedom)). Возвращает счёт плюс сами плоские направления (собственные векторы) — реальные оси, вдоль которых система может двигаться без энергетического штрафа, которые читает модуль поддержки решений.

## §3. Проброс доверия {#доверие}

Всякий скалярный инвариант несёт неопределённость, выведенную из `cov` дельта-методом: для инварианта $f(\Gamma)$, `var(f) = grad_f.T @ cov @ grad_f`, с `grad_f`, вычисленным аналитически где замкнутая форма (P, R, R_phi, D_diff, Meaning) и автодиффом/конечными разностями иначе (Phi, Freedom). Ядро возвращает всякий инвариант как тройку `(value, std, anchor)`. **У входов якоря III нет `cov`**, поэтому их инварианты возвращают `std = null` и отрисовываются как «ориентировочные, не измеренные» — ось честности, проведённая на численном слое.

## §4. Гардрейлы оптической конструкции {#гардрейлы}

Прежде чем всякий отсчёт выпущен, ядро прогоняет три проверки [оптической конструкции](/docs/applied/research/one-grammar#оптическая-конструкция) и прикрепляет вердикты:

1. **Тип карты (T-256).** Какую подконфигурацию это чтение реально разрешает? Опросная оценка разрешает населённости хорошо, а фазы плохо; ядро метит, какие секторы достоверны для *этой* оценки, так что код ниже по течению никогда не читает фазозависимое значение Gap с фазо-слепого якоря.
2. **Лицензия якоря (T-257).** Если какой-либо внешний сигнал питал оценку, на лицензированном ли он реестре (солнечный/лунный/физиологический) или это категориальная ошибка (планетарный/натальный)? Нелицензированные внешние входы отвергаются при приёме, а не тихо используются.
3. **Гейт жизнеспособности.** Реально ли субъект в жизнеспособном режиме ($P>2/7$ и т.д.)? Ниже гейта отсчёты «предиката сознания» помечаются неприменимыми, а не сообщаются как малые числа — у камня нет низкого $\Phi$, у него нет никакого.

Эти вердикты не совещательны; доменный модуль, игнорирующий их, не компилируется против API (§5), потому что тип отсчёта параметризован исходом гардрейла.

## §5. Внутренний API {#api}

Ядро экспонирует одну точку входа и типизированный отсчёт:

```
readout = kernel.evaluate(estimate: GammaEstimate,
                          prev: GammaEstimate | null) -> Readout
```

Поля `Readout`: `invariants` (одиннадцать троек `(value, std, anchor)`; `mode` присутствует только если дан `prev`), `mandalagram` (render-модель), `archetype` (id + отступ), `guardrails` (три вердикта §4), `quality` (флаг проекции входа из §1). Всякое поле, что может быть неопределённым, несёт свою неопределённость; всякое поле несёт свой якорь. **Нет** вызова, возвращающего голое число без его происхождения — API делает нечестный вывод непредставимым.

**Субстрат-агностичность.** `evaluate` никогда не инспектирует, *откуда* пришла `gamma`, кроме тега `anchor`. Тот же вызов обслуживает человеческий самоаудит, ЭЭГ-пайплайн и адаптер ИИ-телеметрии — платформенное свойство [00 §2](/docs/applied/console/overview#почему-коммерция), обеспечиваемое слепотой ядра к субстрату.

## §6. Тестирование и численная дисциплина {#тесты}

Ядро поставляется с оракульной сюитой (та же дисциплина, что использует корпус): аналитические инварианты проверяются против случайных состояний до $\sim10^{-12}$; тождество $R_\varphi = 1-(1-R)^3$ для канонической семьи как кросс-проверка; краевые состояния ($I/7$, чистые состояния) попадают в точные пороговые значения; и property-тесты (монотонность Смысла по всякому фактору, $G_2$-инвариантность всякого скаляра под случайным $G_2$-поворотом осей). Сборка, провалившая любой оракул, не выпускается — авторитет отсчёта держится на том, что ядро доказуемо есть функции корпуса, а не их приближения.

**Куда это ведёт.** [03 · Якоря и оценка](/docs/applied/console/anchors) специфицирует, как реальный `GammaEstimate` производится из каждого из трёх типов свидетельства, с полным инструментом самоаудита и математикой измерительного моста.
